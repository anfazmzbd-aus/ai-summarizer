RFC-0007.6 — DAG-Based Multi-Agent Execution Engine

Status: Draft → Candidate for Adoption
Version: V7.6
Authors: Architecture Working Group
Created: 2026-06-18
Supersedes: V7.5.x
Target Release: V7.6.0

1. Abstract

This RFC defines the architecture, execution semantics, dependency model, and runtime boundaries for the AI Summarizer execution platform.

The system executes analytical agents through a directed acyclic graph (DAG) while explicitly separating preprocessing from execution orchestration.

This specification establishes:

DAG execution rules
scheduler behavior
dependency validation
artifact ownership
execution metadata
parallel execution semantics
2. Motivation

Prior versions mixed preprocessing and execution responsibilities.

Observed issues:

summary treated as executable node
planning mixed with execution
dependency ambiguity
unstable execution ordering
graph contamination

V7.6 introduces strict layer separation.

3. Terminology
Term	Definition
Agent	Executable analytical component
Artifact	Persistent output produced by agents
DAG	Directed acyclic graph of analytical execution
Scheduler	Converts execution order into execution layers
Runtime	State execution and aggregation layer
Preprocessing	Non-DAG transformation phase
Plan	Execution configuration

4.2 Processing Layers
Layer 1 — Request Entry

Responsibilities:

receive request

Primary module:

Layer 2 — Preprocessing (NON-DAG)

Responsibilities:

normalize input

parse sections

Output:

state["summary"]
state["plan"]["sections"]

Modules:

summary_agent
section_parser
Layer 3 — Planning (NON-DAG)

Responsibilities:

detect intent
select execution strategy

Output:

plan["selected_agents"]

Modules:

semantic_router.py
Layer 4 — DAG Engine

Responsibilities:

resolve dependencies
validate graph
build schedule

Modules:

dependency_resolver.py
graph_validator.py
scheduler.py
parallel_executor.py
Layer 5 — Runtime

Responsibilities:

execute
merge artifacts
emit metadata

Modules:

executor.py
state_merger.py
5. DAG Model
5.1 DAG Membership

Only analytical agents SHALL participate.

Allowed:

insights
findings
sentiment
trend
risk
forecast
root_cause
recommendation
actions

Forbidden:

summary
plan
section_parser
semantic_router
5.2 Dependency Rules

Dependencies SHALL be declared:

@register_agent(
    depends_on=[]
)

Rules:

dependency MUST exist
dependency MUST execute first
cycles MUST fail validation
6. Registry

Registry SHALL remain authoritative.

Definition:

AGENT_REGISTRY={}

Registration:

register_agent()

Schema:

{
 "function": fn,
 "depends_on": [],
 "produces": []
}

execution_order

8. Scheduler

Scheduler SHALL:

receive execution_order
construct parallel layers
preserve dependency guarantees

Input:

execution_order
registry

Output:

parallel_groups

Example:

[
 ["insights","findings"],
 ["risk","trend"],
 ["forecast"],
 ["recommendation"]
]
9. Parallel Execution

Execution semantics:

agents in same group MAY execute concurrently
groups MUST execute sequentially

Guarantees:

group(N)
completed
before
group(N+1)

11. Validation

Validator SHALL enforce:

Unknown Agent
execution ⊆ registry
Dependency Ordering
dependency before dependent
DAG Purity

Reject:

summary
plan
section_parser

inside execution_order.

Runtime SHALL emit:

execution_metadata

 "timings":{},

}
